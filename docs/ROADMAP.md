# Development Workflow

For every task:

1. Read MASTER_SPEC.md.
2. Read UI_UX_GUIDELINES.md.
3. Read BLOCKS_SPEC.md.
4. Continue from the first unfinished task.
5. Implement only one feature at a time.
6. Test it.
7. Fix issues.
8. Update documentation.
9. Commit changes.
10. Move to the next task.

Never skip unfinished work.

Never rewrite working code without reason.

Always preserve architecture consistency.

---

# Definition of Ready

A task may begin only when:

* Requirements are understood.
* Dependencies are complete.
* Design is defined.

---

# Definition of Done

A task is complete only if:

* Feature implemented.
* UI polished.
* Responsive.
* Tested.
* No TypeScript errors.
* No ESLint errors.
* No duplicated code.
* Documentation updated.
